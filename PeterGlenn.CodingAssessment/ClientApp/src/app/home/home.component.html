TA Instruments Code Challenge

Summary

This is Peter Glenn's solution for the TA Instruments Code Challenge. The backend is written using .NET Core. The frontend is written using Angular.

There are four pages in the UI:

  • Home: Contains a summary of the solution and the various source files created.
  • Instructions: Contains the Code Challenge instructions that were used as the requirements for the implmented solution.
  • Solution: Contains the working solution based on the provided instructions.
  • Bonus: A modified version of the solution that calculates the Scrabble value for each word in the results list.

Source Code

The source code can be accessed two ways:

  • GitHub Repository - The source code is hosted in a public GitHub Repository.
    • master: Contains the source code without the Bonus functionality.
    • branch (Bonus-ScrabbleValues): Contains the source code with the Bonus Scrabble functionality.
  • Google Drive Zip File - Zip files containing the source code can be downloaded from the shared Google Drive folder.

Source Files

There are two projects in the solution. PeterGlenn.CodingAssessment contians all the frontend and backend functionality. PeterGlenn.CodingAssessment.UnitTests contains the unit tests for the backend functionality.

The majority of the files in the solution were added by the Visual Studio template for a .NET Core Angular app. Below is a list of the files that I added for this solution as well as a list of the existing files that I modified.

Added Files - PeterGlenn.CodingAssessment

  • \ClientApp\src\app\home - Contains the simple HTML and TypeScript files for the Home page.
  • \ClientApp\src\app\instructions - Contains the simple HTML and TypeScript files for the Instructions page.
  • \ClientApp\src\app\words - Contains the HTML and TypeScript files for the Solution page. This is the actual coding challenge solution UI.
  • \ClientApp\src\app\words-scrabble - Contains the HTML and TypeScript files for the Bonus page. Only present in the Bonus-ScrabbleVaues branch.
  • \Controller\WordsController.cs - The controller that exposes the backend endpoints for retrieving the matching words. These are the api endpiont the UI calls when it wants to get the list of matching words.
  • \Application\WordsApplication.cs - This is the main "Business Logic" class that contains the logic/algorithms for finding all the matcing words using the letters provided.
  • \Repositories\WordsRepositories.cs - This class reads the sowpods.txt file and provides the word list to the application. Also provides the Scrabble letter values in the Bonus-ScrabbleValues branch.
  • \Resources\sowpods.txt - The provided word list to be used in the code challenge.

Added Files - PeterGlenn.CodingAssessment.UnitTests

  • WordsControllerTests.cs - Units tests for the WordsController class. All tests pass.
  • WordsApplicationTests.cs - Units tests for the WordsApplication class. All tests pass.

Modified Files - PeterGlenn.CodingAssessment

  • \ClientApp\src\app\nav-menu - Changes to navigation menu to add the menu options.
  • \ClientApp\src\app\app.module.ts - Changes to register the new angular components to the angular application.
  • Startup.cs - Register the WordsController and WordsApplication in the IoC Container to support a testable code base.
